# Development

## Run the full stack locally (Supabase-first)

This project now uses Supabase for local data/auth/storage instead of Firebase database emulators.

### 1) Prerequisites

- Docker Desktop (running)
- Supabase CLI
- Node.js 20.x
- Python 3.10+

### 2) One-time setup

From the repository root:

```bash
supabase init
supabase link --project-ref <your-project>
supabase start
```

What this gives you locally:

- Postgres
- Auth
- Storage
- Supabase Studio

Use `supabase link --project-ref <your-project>` when you want to sync/push/pull against a remote Supabase project.

### 3) Environment setup

- Copy `.env.example` to `.env`.
- Set:

```env
SUPABASE_URL=http://localhost:54321
```

- Use the local anon key generated by `supabase start` (from the local Supabase env output/files) in your `.env`.

### 4) Running services

#### Functions (Supabase Edge Functions)

Use Supabase Edge Functions as the runtime (no Firebase host/emulator):

```bash
cd ai-volunteer-mvp
supabase functions serve whatsapp-webhook --env-file ../.env
supabase functions serve volunteer-response --env-file ../.env
supabase functions serve need-created --env-file ../.env
```

Invoke locally with:

```bash
curl -i --location 'http://127.0.0.1:54321/functions/v1/whatsapp-webhook' \
  --header 'Content-Type: application/json' \
  --data '{"Body":"Emergency water needed near Central Park","From":"+1234567890"}'
```

#### Python prediction service

```bash
source venv/bin/activate
python ai-volunteer-mvp/prediction-service/needPredictionService.py
```

Windows PowerShell activation:

```powershell
.\venv\Scripts\Activate.ps1
python ai-volunteer-mvp/prediction-service/needPredictionService.py
```

#### Dashboard (Vite)

```bash
npm run dev
```

#### Expo PWA

```bash
npx expo start
```

### 5) Testing workflow

- Inspect local data/auth/storage in Supabase Studio: http://localhost:54323
- Test inbound webhooks with curl or Postman.
- Run test suites:

```bash
cd ai-volunteer-mvp/functions
npm test
```

```bash
cd ai-volunteer-mvp/prediction-service
pytest
```

CI note (GitHub Actions):

- You can run integration tests against a local Supabase stack in CI by installing Supabase CLI, running `supabase start`, then executing Jest/pytest.
- Deploy edge functions in CI with:

```bash
supabase functions deploy whatsapp-webhook
supabase functions deploy volunteer-response
supabase functions deploy need-created
```

### 6) Reset local DB

```bash
supabase db reset
```

This drops local DB state and re-runs migrations from scratch.

### 7) Troubleshooting

- Docker port conflicts:
  - Stop services using Supabase ports, then re-run `supabase start`.
  - Common conflicts are caused by existing Postgres/docker containers.
- View logs:

```bash
supabase logs
```

- RLS blocks local queries:
  - Preferred: add temporary local dev policies.
  - Last-resort for short testing sessions only: disable RLS on specific tables, then re-enable.

```sql
ALTER TABLE public.needs DISABLE ROW LEVEL SECURITY;
-- run your local test
ALTER TABLE public.needs ENABLE ROW LEVEL SECURITY;
```
